# Frank Design Principle

このドキュメントはFrankのデザイン品質の「床」を定義する。
案件固有の世界観は各プロジェクトの `design-brief.md` で別途定義し、このドキュメントは上書きしない。

---

## 1. タイポグラフィ

- フォントサイズは明確な階層を持つ（最低3段階：見出し / 本文 / 補足）
- 見出しと本文のサイズ差は1.5倍以上を目安にする
- フォントウェイトは700（Bold）と400（Regular）の2択を基本とし、必要に応じて500（Medium）を追加
- 行間（line-height）は本文1.6以上、見出し1.2前後
- 1画面に使うフォントファミリーは原則1種類。最大2種類まで
- **句読点（「、」「。」）は使わない。区切りは半角スペース（ ）で表現する**

### 確定：lineHeightとタイトル-リード間隔（2026-06-27 ラボ先輩目視確認）

| 要素 | lineHeight |
|------|------------|
| タイトル | **1.2** |
| リード文 | **1.4** |

- タイトルとリード文の間隔（marginBottom）は、各行間の最大値より広くする
  - タイトル行間 = fontSize × (1.2-1)、リード文行間 = fontSize × (1.4-1)
  - この両方より広い値を marginBottom に設定する（目安: リード文fontSize × 0.8 以上）

### 確定：改行ルール（2026-06-27）

- **ワードの途中で改行しない**（日本語: `word-break: keep-all`、英語: デフォルトで保護済み）
- **行ごとの文字数バランスを均等に保つ**（1行目15文字・2行目2文字のような極端な差はNG）
  - CSSで自動解決: `text-wrap: balance`（Remotion/Chromium環境で動作）
  - 自動で解決しない場合は手動で `<br />` を入れてバランスを調整する
- **自動折り返しでもバランスが偏る場合の対処（優先順）：**
  1. テキストフィールドの横幅を広げて1行に収める
  2. それでも2行になるなら、文節の良い区切りで明示的に `<br />` を入れてバランスを取る
  3. 「折り返したら短い行が残る」という状態を放置しない
- 改行が必要なテキストを書く際、俺は**必ず行バランスを目視確認してから確定する**

### 確定：FHD(1920x1080)におけるフォントサイズ範囲（2026-06-27 ラボ先輩目視確認）

| 要素 | 最大 | 最小（暫定） |
|------|------|-------------|
| タイトル | **112px** / weight:700 | **48px** / weight:700 |
| リード文 | **48px** / weight:400 | **28px** / weight:400 |
| Eyebrow | — | **24px** / weight:400（リード文最小値より一段下） |

### 確定：Eyebrow（アイブロウ）テキストの定義（2026-06-27 ラボ先輩目視確認）

**Eyebrowとは：** 「STEP 1」「Chapter 2」のような、流れや順序を示す**補助ラベル**。
「しっかり読む」ではなく「感じる程度」でよい情報に使う。

**使い分けの判断基準（Frank自律判断）：**
- 情報の本質・内容を伝える → **Heading**（しっかり読ませる）
- 順序・分類・補足を示すラベル → **Eyebrow**（感じる程度でOK）

**Eyebrowの視覚的特徴（確定値）：**

| 属性 | Eyebrow | Heading |
|------|---------|---------|
| フォントサイズ | **24px**（Headingの60%以下が目安） | 案件に応じる |
| fontWeight | **400** | **700** |
| 色のopacity | **40〜50%**（薄く存在感を下げる） | **100%** |
| letter-spacing | **2〜3px**（広めで軽さを演出） | 通常（0〜0.5px） |

### 確定：法務テキスト仕様（コピーライト・注釈・引用元）（2026-06-27 ラボ先輩目視確認）

| 項目 | 値 |
|------|-----|
| フォントサイズ | **16px** |
| fontWeight | **300**（細め。つぶれ防止） |
| 色（黒背景時） | `rgba(255,255,255,0.55)`（純白ベース） |
| 色（白背景時） | `rgba(0,0,0,0.45)`（濃いグレー） |
| 下からの距離 | **25px** |
| 右マージン（右下配置時） | **50px** |

**配置パターン（案件に応じて選択）：**
- **左下**：注釈（1行目）＋ コピーライト（2行目）
- **センター**：コピーライトのみ
- **右下**：コピーライトのみ（right: 50px）

**法務テキストの自動判断ルール（Frank自律判断）：**
- 資料に数値・統計・調査結果が含まれる → 「※数値は概算/出典元」注釈を左下に追加
- クライアント名・商標が含まれる → コピーライト表記を追加
- 外部データ・引用が含まれる → 「出典：○○」を左下に追加
- 完全にオリジナルのビジュアルのみ → 法務テキスト不要

## 2. カラー

- 1案件で使うカラーは原則3色以内（Primary / Accent / Neutral）
- 純色（red, blue, green等）は使わない。必ずHSLで彩度・明度を調整する
- **原色に近すぎる強い色は避ける**。彩度（S）は概ね65%以下を目安にする
  - NG例：`hsl(0, 100%, 50%)` = 純赤 / `#cc3333`（彩度60%）も高め
  - OK例：`hsl(0, 45%, 42%)`（落ち着いた赤）のように彩度・明度を抑える
- 背景と文字のコントラスト比は最低4.5:1を確保する
- グラデーションを使う場合、色相差は30°以内を推奨（離れすぎると安っぽくなる）
- 透明度（opacity）で色を作る場合、背景色が変わると崩れることを意識する

## 3. スペーシング・レイアウト

- 余白の基準単位は8pxとし、倍数（8, 16, 24, 32, 48, 64）で統一する
- 要素間の余白は「詰めすぎ」よりも「空けすぎ」の方がマシ
- 情報密度が高い場合はカードや区切り線で視覚的なグループを作る
- 中央揃えを多用しない。左揃えを基本とし、中央揃えはタイトルや強調要素のみ

### 確定：FHD(1920x1080)における画面端余白（2026-06-27 ラボ先輩目視確認）

| 方向 | 値 | 備考 |
|------|-----|------|
| 上   | **65px**  | |
| 左右 | **82px**  | |
| 下   | **123px** | 字幕スペースを考慮して上より大きめ |

- 要素間のGAPは32pxを基本とする（未確定、今後調整予定）

### 確定：オブジェクト間GAPの指針（2026-06-27 ラボ先輩目視確認）

- **基本値: GAP = 32px**
- 複数のオブジェクトを画面内に整列させる場合（グリッドレイアウト等）は GAP < 画面端余白が自然に見えやすい
- ただし「常に」ではない。デザイン意図によっては GAPを広げる場合もある
- 大小関係の参考値: `GAP(32px) < PAD_TOP(65px) < PAD_SIDE(82px) < PAD_BOTTOM(123px)`

### 確定：オブジェクト内テキストのルール（2026-06-27）

- **改行が優先**：テキストが長い場合はまず改行で対応する。シェイプのサイズは変えない
- **横幅拡張は苦肉の策**：折り返しで行バランスが著しく偏る場合は、テキストフィールドの横幅を広げるか 文節の良い部分で明示的に改行する
- **同じ役割・同じ階層のテキストが複数並ぶとき、1つだけ際立って長い場合は改行して長さを均等に揃える**
  - 例：「デントリペア / AIによる再スキャン / 品質チェック / 納車」→ 「AIによる再スキャン」だけ長いので改行
  - 他のテキストと視覚的な幅が揃うことで、並列感・統一感が生まれる
- **`calcMinWidth` ヘルパーで必要幅を自動計算してminWidthに設定する（折り返し自動防止）：**
  ```ts
  // 日本語全角文字 ≒ fontSize px。係数1.1で余裕を確保
  const calcMinWidth = (lines: string[], fontSize: number, padX: number) =>
    Math.ceil(Math.max(...lines.map(l => [...l].length)) * fontSize * 1.1) + padX * 2;
  ```
- テキストは `行ごとの配列` で管理し、各行を `<div>` でレンダリング（`<br />` より管理しやすい）
- `word-break: keep-all` を全テキスト要素に設定（文字途中の折り返し防止）
- オブジェクト内のpaddingは常に固定値を守る。テキストがpaddingに食い込むのはNG

**丸（Circle）のスケールルール：**
- 丸は正円を維持する。横幅を広げる場合は縦も同じ数値で広げる（width = height を常に保つ）

**シェイプサイズ統一ルール：**
- 同一グループ内でシェイプのサイズを変更した場合、**同じ役割・同じサイズのシェイプは全て同じ寸法にする**
- 位置（座標）は各シェイプ独立で構わない

## 4. アニメーション

- 登場アニメーションは `easeOut` または `cubicBezier(0.16, 1, 0.3, 1)` を基本とする
- duration は短め（300〜600ms）が上品。1秒を超えるものは理由が必要
- 複数要素を順番に出す場合、staggerは50〜100ms程度
- ループアニメーションは微細（scale 1.0→1.02程度）にとどめ、主役を邪魔しない
- アニメーションは「ない方がいい」場合もある。動かすことを目的化しない

---

## 5. NG集（やってはいけないパターン）

- ドロップシャドウを多用しない（1画面に1〜2箇所が限界）
- テキストに複数色を混ぜない（強調は太字か大きさで表現する）
- アニメーションのovershoot（行き過ぎてから戻る）は意図的でない限り使わない
- グラデーションの色数を3色以上にしない
- フォントサイズが9種類以上にならないようにする

## 6. アイコン

### 確定：アイコンの描き方原則（2026-06-27）

- **形式は常にPNG**
- **ユニバーサルデザイン的なシンプルさ**：線が少なく 誰が見ても意味がわかる形
- 細かすぎる描写や装飾は避ける

**背景色と合成モードの使い分け：**

| アイコンのメインカラー | 背景 | 合成モード | 効果 |
|---|---|---|---|
| 暗い（黒・濃色） | 単色の白 | **乗算（Multiply）** | 白が消えてアイコンが残る |
| 明るい（白・淡色） | 単色の黒 | **スクリーン（Screen）** | 黒が消えてアイコンが残る |

- CSS実装: `mix-blend-mode: "multiply"` または `"screen"`
- アイコンサイズは**シェイプ内側に視覚的な余白が見える**範囲で収める（比率固定はしない）
  - シェイプが小さい → アイコンを相対的に大きく取っても窮屈に見えない
  - シェイプが大きい → アイコンを絶対値でキャップして、余白を多めに確保する
- アイコンはシェイプ内に中央配置（`alignItems: center / justifyContent: center`）
- **アイコンのトーンはデザイン全体のカラートーンに合わせる**
  - 純粋な黒（`#000`）はそのまま使わない。周囲の色が落ち着いていればアイコンも合わせる
  - 実装：Layer 3コンテナに `opacity: 0.6〜0.8` をかけてアイコンのトーンを馴染ませる
  - multiply合成との組み合わせでは、opacityは白（背景透過）に影響しないため安全に使える
